# 文档智能比对分析工具 - 功能演示

## 📖 演示场景

假设您需要比对一份合同的两个版本，以下是完整的使用流程演示。

---

## 🎬 演示步骤

### 第一步：上传文档

#### 操作说明
1. 打开应用，看到标题 **"文档智能比对分析工具"**
2. 页面顶部有两个上传区域：
   - 左侧：**上传文档 1（原始版本）**
   - 右侧：**上传文档 2（修改版本）**

#### 上传方式
**方式一：拖拽上传**
- 将合同原始版本拖拽到左侧上传区域
- 将合同修改版本拖拽到右侧上传区域

**方式二：点击上传**
- 点击左侧上传区域，选择原始版本文件
- 点击右侧上传区域，选择修改版本文件

#### 上传后显示
```
📄 合同_v1.docx
   256 KB
```

---

### 第二步：开始比对

#### 操作说明
1. 确认两个文档都已上传成功
2. 点击中间的蓝色按钮 **"开始比对分析"**
3. 按钮显示 **"正在比对..."** 和旋转动画

#### 比对过程
- 系统解析文档内容（约 2-5 秒）
- 执行差异比对（约 1-3 秒）
- 生成差异结果
- 自动触发 AI 分析

#### 完成提示
弹出 Toast 消息：
```
✅ 比对完成
检测到 15 处新增内容（约 120 个词），8 处删除内容（约 65 个词）。
```

### 第五步：查看 AI 详细分析

#### AI 分析报告结构

```markdown
# 📊 文档差异分析报告

## 一、差异概览

本次合同修订主要涉及违约责任、保密义务和争议解决等关键条款的调整。
整体变更幅度为中等，新增内容主要集中在风险控制和责任界定方面。

## 二、逐项差异分析

### 2.1 新增内容分析

**差异点 1：新增违约责任条款**
- **内容摘要**：第三条增加了关于违约责任的详细说明
- **变更类型**：新增条款
- **重要程度**：⭐⭐⭐⭐⭐
- **影响分析**：明确了双方违约的责任和赔偿标准，有利于减少纠纷
- **建议**：建议双方仔细审阅违约责任的具体金额和计算方式

**差异点 2：新增保密义务条款**
- **内容摘要**：第七条新增了关于保密义务的详细规定
- **变更类型**：新增条款
- **重要程度**：⭐⭐⭐⭐
- **影响分析**：加强了商业秘密保护，降低了信息泄露风险
- **建议**：建议明确保密期限和违反保密义务的后果

**差异点 3：新增技术规格附件**
- **内容摘要**：新增附件一：技术规格说明
- **变更类型**：新增附件
- **重要程度**：⭐⭐⭐⭐
- **影响分析**：使技术要求更加明确，便于执行和验收
- **建议**：建议技术部门仔细审核技术规格的可行性

### 2.2 删除内容分析

**差异点 1：删除原合同期限条款**
- **内容摘要**：删除了原第三条关于合同期限为一年的规定
- **变更类型**：删除条款
- **重要程度**：⭐⭐⭐⭐⭐
- **影响分析**：可能改为其他期限或改为长期合同，需要特别关注
- **建议**：务必确认新的合同期限安排

**差异点 2：删除原争议解决条款**
- **内容摘要**：删除了原第五条关于争议解决的简单描述
- **变更类型**：删除条款
- **重要程度**：⭐⭐⭐⭐
- **影响分析**：可能在其他位置重新规定或采用新的争议解决方式
- **建议**：确认新版本中是否有替代的争议解决条款

## 三、整体变更分析

### 3.1 变更主题
本次修订的主要主题是**强化风险控制和责任界定**，通过新增违约责任、
保密义务等条款，使合同更加完善和严谨。

### 3.2 变更程度
整体变更程度为**中等**。虽然新增和删除的条款数量不多，但涉及的都是
合同的核心条款，对双方权利义务有重要影响。

### 3.3 变更影响
- **对甲方的影响**：责任和义务更加明确，但同时也受到更多约束
- **对乙方的影响**：权益保护更加完善，但需要承担更多保密义务
- **对合同执行的影响**：技术规格更明确，有利于合同的顺利执行

### 3.4 风险提示
⚠️ **重点关注**：
1. 原合同期限条款被删除，务必确认新的期限安排
2. 违约责任金额较高，需要评估风险承受能力
3. 保密义务的范围和期限需要明确界定
4. 争议解决条款的变更可能影响纠纷处理方式

## 四、总体评价与建议

### 4.1 变更合理性评价
本次变更总体上是**合理且必要的**。新增的条款有助于完善合同体系，
明确双方权利义务，降低合同执行风险。

### 4.2 改进建议
1. 建议补充明确的合同期限条款
2. 建议在违约责任条款中增加免责情形的说明
3. 建议在保密义务条款中明确保密期限
4. 建议增加合同变更和解除的程序性规定

### 4.3 注意事项
1. ⚠️ 签署前务必确认所有关键条款都已明确
2. ⚠️ 建议法务部门对新增条款进行专业审查
3. ⚠️ 建议技术部门对技术规格附件进行可行性评估
4. ⚠️ 建议财务部门评估违约责任金额的合理性
```

## 💡 实际应用场景

### 场景一：法务审查
**需求**：法务部门需要审查合同修订版本

**操作流程**：
1. 上传原合同和修订版本
2. 查看差异导航，快速定位所有变更
3. 逐个查看关键条款的变更
4. 参考 AI 的风险提示和建议
5. 重点审查高重要度的变更
6. 导出分析报告，提交审查意见

### 场景二：项目经理审阅
**需求**：项目经理需要了解项目文档的修订内容

**操作流程**：
1. 上传项目文档的两个版本
2. 查看 AI 的差异概览，了解整体变更
3. 使用差异导航快速浏览所有变更
4. 重点查看与项目执行相关的变更
5. 评估变更对项目进度和成本的影响

### 场景三：技术文档对比
**需求**：技术人员需要对比技术规格的变更

**操作流程**：
1. 上传技术规格的两个版本
2. 使用差异导航定位技术参数的变更
3. 对照查看参数变更的具体内容
4. 参考 AI 分析，评估技术可行性
5. 确认变更是否符合技术要求
